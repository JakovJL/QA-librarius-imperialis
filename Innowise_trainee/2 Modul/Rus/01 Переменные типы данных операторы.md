## Примитивные типы данных

В Java 8 примитивных типов. Они хранят значения напрямую в памяти.

| Тип | Размер | Диапазон | По умолч. | Пример |
|---|---|---|---|---|
| `byte` | 1 байт | -128 до 127 | 0 | `byte b = 100;` |
| `short` | 2 байта | -32 768 до 32 767 | 0 | `short s = 1000;` |
| `int` | 4 байта | ~-2.1 млрд до 2.1 млрд | 0 | `int age = 25;` |
| `long` | 8 байт | очень большой | 0L | `long pop = 8_000_000_000L;` |
| `float` | 4 байта | ~±3.4E+38 | 0.0f | `float pi = 3.14f;` |
| `double` | 8 байт | ~±1.7E+308 | 0.0 | `double price = 19.99;` |
| `boolean` | 1 бит | true / false | false | `boolean isReady = true;` |
| `char` | 2 байта | один символ Unicode | '\u0000' | `char grade = 'A';` |

> [!tip] Нижние подчёркивания в числах
> Можно использовать `_` для читаемости: `int million = 1_000_000;` Java игнорирует подчёркивания.

```java
public class PrimitiveDemo {
    public static void main(String[] args) {
        int age = 25;
        double price = 19.99;
        boolean isStudent = true;
        long worldPopulation = 8_000_000_000L;
    }
}
```

## Ссылочные типы

Хранят **адрес в памяти** на объект, а не само значение.

| Тип | Описание | Пример |
|---|---|---|
| `String` | Текст (неизменяемый) | `String name = "Alice";` |
| Массивы | Коллекция одного типа | `int[] numbers = {1, 2, 3};` |
| Классы | Пользовательские типы | `Scanner input = new Scanner(System.in);` |

### Примитивы vs Ссылки

```java
// Примитив — копирует ЗНАЧЕНИЕ
int a = 5;
int b = a;
b = 10;         // a всё ещё 5

// Ссылка — копирует АДРЕС (оба указывают на один объект)
int[] arr1 = {1, 2, 3};
int[] arr2 = arr1;
arr2[0] = 99;   // arr1[0] теперь тоже 99!
```

> [!warning] Ловушка ссылок
> Когда присваиваешь одну ссылочную переменную другой, они указывают на один объект. Изменение через одну затрагивает другую.

## Объявление и инициализация переменных

```java
int count = 10;                     // 1. Объявление + инициализация
int count; count = 10;              // 2. Раздельно
int x = 1, y = 2, z = 3;           // 3. Несколько одной строкой

int value;
// System.out.println(value);       // ❌ Ошибка: не инициализирована
value = 42;
System.out.println(value);          // ✅ OK
```

## Приведение типов (casting)

### Неявное (расширение) — безопасно, автоматически

```java
int myInt = 100;
long myLong = myInt;      // int → long, OK
```

**Порядок:** `byte → short → int → long → float → double`

### Явное (сужение) — может потерять данные

```java
double price = 19.99;
int rounded = (int) price;     // 19 (дробная часть отброшена!)
```

> [!caution] Потеря данных
> `(int) 19.99` даёт `19`. Десятичная часть пропадает без предупреждения.

## Операторы

### Арифметические

| Оператор | Значение | Пример |
|---|---|---|
| `+` | Сложение | `5 + 3` → `8` |
| `-` | Вычитание | `5 - 3` → `2` |
| `*` | Умножение | `5 * 3` → `15` |
| `/` | Деление | `10 / 3` → `3` (целое!) |
| `%` | Остаток | `10 % 3` → `1` |

> [!warning] Деление целых чисел
> `10 / 3` даёт `3`, а не `3.33`. Используй `10 / 3.0` для `3.333...`.

### Сравнения и логические

| Оператор | Значение |
|---|---|
| `==` `!=` | Равно / Не равно |
| `>` `<` `>=` `<=` | Больше, меньше и т.д. |
| `&&` `||` `!` | И, ИЛИ, НЕ |

> [!caution] `==` для String
> Используй `.equals()` для сравнения содержимого: `"hello".equals("hello")` → `true`.

## Основы String

```java
String name = "Alice";
System.out.println(name.length());                  // 5
System.out.println(name.charAt(0));                // 'A'
System.out.println(name.substring(1, 4));          // "lic"
System.out.println(name.equals("Alice"));          // true
System.out.println(name.toUpperCase());            // "ALICE"
System.out.println(name.contains("lic"));          // true
```

> [!info] Неизменяемость строк
> Строки нельзя изменить после создания. Каждое "изменение" создаёт новый объект. Для активной работы используй `StringBuilder`.

## var (Java 10+)

Компилятор определяет тип сам. Переменная всё ещё строго типизирована.

```java
var name = "Alice";          // String
var age = 25;                // int
var numbers = new int[5];    // int[]

// var name;                 // ❌ Нужно сразу инициализировать
```

> [!tip] Когда использовать
> Используй, когда тип очевиден: `var list = new ArrayList<String>();`. Избегай, когда непонятно: `var result = someMethod();`.

## Классы-обёртки и автоупаковка

Дженерики и коллекции не работают с примитивами (`List<int>` недопустим). Java предоставляет **класс-обёртку** для каждого примитивного типа — класс, хранящий примитивное значение как объект.

| Примитив | Обёртка | Пример |
|---|---|---|
| `int` | `Integer` | `Integer num = 42;` |
| `double` | `Double` | `Double pi = 3.14;` |
| `boolean` | `Boolean` | `Boolean flag = true;` |
| `char` | `Character` | `Character ch = 'A';` |
| `long` | `Long` | `Long big = 100L;` |
| `byte` | `Byte` | `Byte b = 1;` |
| `short` | `Short` | `Short s = 10;` |
| `float` | `Float` | `Float f = 2.5f;` |

### Автоупаковка и распаковка

Java конвертирует между примитивами и обёртками автоматически:

```java
// Автоупаковка (autoboxing): примитив → обёртка (автоматически)
Integer num = 42;             // компилятор делает: Integer.valueOf(42)
List<Integer> list = new ArrayList<>();
list.add(5);                  // автоупаковка 5 → Integer.valueOf(5)

// Распаковка (unboxing): обёртка → примитив (автоматически)
int x = num;                  // компилятор делает: num.intValue()
int first = list.get(0);     // авто-распаковка Integer → int
```

### Ловушка кеша Integer

Java кеширует значения `Integer` от **-128 до 127**. В этом диапазоне `Integer.valueOf()` возвращает один и тот же объект. За пределами диапазона — каждый раз создаёт новый.

```java
Integer a = 127;
Integer b = 127;
System.out.println(a == b);    // true — один закешированный объект

Integer c = 128;
Integer d = 128;
System.out.println(c == d);    // false — разные объекты!
System.out.println(c.equals(d)); // true — одинаковое значение
```

> [!danger] Всегда используй .equals() для обёрток
> Никогда не сравнивай обёртки через `==`. Для малых чисел (кеш) работает, для больших — нет. Всегда используй `.equals()`.

### Опасность null

Обёртки могут быть `null`, примитивы — нет. Распаковка `null` выбрасывает `NullPointerException`:

```java
Integer value = null;
int x = value;           // NullPointerException! (пытается вызвать value.intValue())
```

Это очень частый баг при работе с коллекциями, результатами из БД или парсинге JSON — любой из этих источников может вернуть `null` вместо числа.

> [!tip] Когда что использовать
> Используй **примитивы** для локальных вычислений, счётчиков циклов и везде, где `null` не нужен. Используй **обёртки** когда `null` — допустимое значение (например, "данных нет"), в коллекциях (`List<Integer>`) и в обобщённом коде.

## Вопросы для собеседования

### Топ-10

**1. Сколько примитивных типов в Java? Назовите их.**
8 типов: `byte`, `short`, `int`, `long`, `float`, `double`, `boolean`, `char`. Хранят значения напрямую, не объекты.

**2. В чём разница между примитивными и ссылочными типами?**
Примитивы хранят значение в памяти. Ссылочные — адрес на объект. Примитивы сравниваются по значению (`==`). Для ссылочных `==` сравнивает адреса, а `.equals()` — содержимое.

**3. Что такое приведение типов? Когда нужно явное приведение?**
Приведение — конвертация одного типа в другой. Неявное (расширение) — автоматически при переходе от меньшего типа к большему. Явное (сужение) — когда от большего к меньшему, может терять данные.

**4. В чём разница между `==` и `.equals()` для String?**
`==` сравнивает адреса в памяти (указывают ли на один объект). `.equals()` сравнивает содержимое строк. Для сравнения текста всегда используй `.equals()`.

**5. Чему равно `10 / 3` в Java? Почему?**
Результат `3`, а не `3.333`, потому что оба операнда — целые числа (int). Java выполняет целочисленное деление, отбрасывая дробную часть.

**6. В чём разница между `++x` и `x++`?**
`++x` (пре-инкремент) — сначала увеличивает, потом возвращает. `x++` (пост-инкремент) — сначала возвращает текущее значение, потом увеличивает. Если `x = 5`, `++x` вернёт 6, `x++` вернёт 5 (x станет 6).

**7. Что такое var? Когда его использовать?**
`var` (Java 10+) позволяет компилятору самому определить тип переменной. Используй, когда тип очевиден: `var list = new ArrayList<String>();`. Избегай, когда тип непонятен: `var result = getData();`.

**8. Почему String неизменяемый?**
Для безопасности (пароли), кеширования (String pool), потокобезопасности и кеширования хэш-кода. Каждое "изменение" создаёт новый объект String.

**9. Чем отличается float от double?**
`float` — 32 бита, точность ~7 знаков, нужен суффикс `f`. `double` — 64 бита, точность ~15 знаков, используется по умолчанию для дробных чисел.

**10. Что такое составные операторы присваивания?**
Операторы, совмещающие операцию с присваиванием: `+=`, `-=`, `*=`, `/=`, `%=`. Пример: `x += 5` равно `x = x + 5`. Они также делают неявное приведение типов.

### Хитрые вопросы

**1. Чему равно `2.0 - 1.1` в Java? Почему?**
Не `0.9`. Будет `0.8999999999999999` из-за погрешности floating-point. Двоичное представление не может точно хранить `0.1`. Для точных денежных расчётов используй `BigDecimal`.

**2. Можно ли использовать var для параметров методов?**
Нет. `var` работает только для локальных переменных внутри методов. Нельзя использовать для полей, параметров, возвращаемых типов.

**3. Почему у Java нет примитивного типа для строк?**
String сделали классом для гибкости: методы (`length()`, `substring()`), String pool, иммутабельность. Примитивы не могут иметь методов.

**4. Что будет при касте большого long в int?**
Если значение выходит за диапазон int, биты обрезаются, и результат заворачивается (overflow). Исключения не будет. Например, `(int) 3_000_000_000L` даст отрицательное число.

**5. Можно ли изменить строку после создания?**
Нет — String immutable. Технически можно через reflection, но это ломает контракт. Для изменяемых строк используй `StringBuilder`.
