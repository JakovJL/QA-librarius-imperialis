## Почему файловый ввод-вывод важен

Программе часто нужно работать не только с памятью, но и с файлами на диске. Например:

- читать конфиги
- сохранять логи
- загружать тестовые данные из JSON или CSV
- писать отчёты
- проверять, существует ли скачанный файл

Это называется **File I/O**, где:

- `I` означает input
- `O` означает output

В Java работа с файлами обычно делается через пакет `java.nio.file` и классы потоков/reader/writer.

## Path File и Files

В современной Java обычно предпочитают `Path` и `Files` из `java.nio.file`.

### `Path`

`Path` представляет путь к файлу или директории.

```java
Path path = Path.of("data", "users.txt");
System.out.println(path); // data\users.txt or data/users.txt
```

Так можно строить пути платформенно-независимым способом, не прописывая разделители вручную.

### `Files`

`Files` — это utility-класс с частыми операциями над путями.

```java
Path path = Path.of("data/users.txt");

System.out.println(Files.exists(path));
System.out.println(Files.isRegularFile(path));
System.out.println(Files.isDirectory(path));
```

### Старый `File`

До NIO.2 Java использовала `java.io.File`.

```java
File file = new File("data/users.txt");
System.out.println(file.exists());
```

`File` до сих пор встречается в старом коде, но для нового кода `Path` и `Files` обычно понятнее и мощнее.

> [!tip] Выбор по умолчанию
> В современной Java начинай с `Path` и `Files`. `File` тоже нужно знать, потому что в старых проектах он всё ещё встречается.

## Чтение файлов

### Прочитать весь текст

```java
Path path = Path.of("data/config.txt");
String text = Files.readString(path);
System.out.println(text);
```

Это удобно для небольших текстовых файлов.

### Прочитать все строки

```java
List<String> lines = Files.readAllLines(Path.of("data/users.txt"));
lines.forEach(System.out::println);
```

Здесь каждая строка файла становится отдельным элементом списка.

### Буферизованное чтение

Для больших файлов лучше использовать `BufferedReader`.

```java
try (BufferedReader reader = Files.newBufferedReader(Path.of("data/users.txt"))) {
    String line;
    while ((line = reader.readLine()) != null) {
        System.out.println(line);
    }
}
```

### Чтение байтов

```java
byte[] bytes = Files.readAllBytes(Path.of("image.png"));
System.out.println(bytes.length);
```

Это используется для бинарных файлов: изображений, PDF, архивов и т.д.

## Запись файлов

### Записать весь текст

```java
Path path = Path.of("output/result.txt");
Files.writeString(path, "Test passed");
```

Если файл уже существует, его содержимое по умолчанию заменяется.

### Записать строки

```java
List<String> report = List.of("Test 1: PASS", "Test 2: FAIL");
Files.write(Path.of("output/report.txt"), report);
```

### Добавить в конец файла

```java
Files.writeString(
    Path.of("logs/app.log"),
    "New log line\n",
    StandardOpenOption.CREATE,
    StandardOpenOption.APPEND
);
```

### Создать директории

```java
Path dir = Path.of("output/reports");
Files.createDirectories(dir);
```

Это удобно перед сохранением скриншотов или отчётов.

> [!warning] Родительская папка должна существовать
> Если пытаться записать файл в папку, которой нет, Java может выбросить исключение. При необходимости сначала создай директории.

## Байтовые потоки и символьные потоки

В Java есть две большие группы I/O-классов.

### Байтовые потоки

Используются для бинарных данных.

Частые классы:

- `InputStream`
- `OutputStream`
- `FileInputStream`
- `FileOutputStream`

Пример:

```java
try (InputStream in = new FileInputStream("photo.png")) {
    int b;
    while ((b = in.read()) != -1) {
        // process byte
    }
}
```

### Символьные потоки

Используются для текстовых данных.

Частые классы:

- `Reader`
- `Writer`
- `FileReader`
- `FileWriter`
- `BufferedReader`
- `BufferedWriter`

Пример:

```java
try (BufferedWriter writer = Files.newBufferedWriter(Path.of("output.txt"))) {
    writer.write("Hello");
}
```

### Что когда использовать

- Используй байтовые потоки для изображений, PDF, архивов, исполняемых файлов
- Используй символьные потоки для текстовых файлов вроде `.txt`, `.csv`, `.json`, `.xml`

> [!info] Кодировка важна
> Текст на диске хранится как байты. Java преобразует байты в символы через кодировку, например UTF-8. Если кодировка выбрана неверно, текст может отображаться сломанно.

## Try-With-Resources и ошибки файлов

Работа с файлами часто использует ресурсы, которые нужно закрывать. Лучший вариант — **try-with-resources**.

```java
try (BufferedReader reader = Files.newBufferedReader(Path.of("data.txt"))) {
    System.out.println(reader.readLine());
} catch (IOException e) {
    e.printStackTrace();
}
```

### Почему это помогает

- автоматически закрывает файл
- код короче
- безопаснее, чем ручное закрытие в `finally`

### Частые исключения

- `IOException`
- `FileNotFoundException`
- `NoSuchFileException`
- `AccessDeniedException`

### Типичные проблемы

- неверный путь
- отсутствующий файл
- нет прав доступа
- файл заблокирован другим процессом
- неверная кодировка

> [!caution] Relative paths
> Относительные пути зависят от рабочей директории приложения. В тестах это легко путает. Если есть проблема с файлами, всегда проверяй, из какой папки реально запускается программа.

## Файловый ввод-вывод в AQA

Файловые операции очень часто встречаются в автотестировании.

### Типичные случаи

- чтение тестовых данных из JSON, CSV или properties
- сохранение API-ответов для отладки
- проверка скачанных файлов
- запись пользовательских логов
- сохранение скриншотов и вложений

### Пример: чтение тестовых данных

```java
String json = Files.readString(Path.of("src/test/resources/data/user.json"));
System.out.println(json);
```

### Пример: проверка скачанного файла

```java
Path downloaded = Path.of("downloads/report.pdf");
if (!Files.exists(downloaded)) {
    throw new AssertionError("Downloaded file not found");
}
```

### Хорошие практики

- храни тестовые данные в `src/test/resources`
- используй `Path.of(...)` вместо хардкода разделителей
- используй try-with-resources
- не хардкодь абсолютные пути без необходимости
- очищай временные файлы, если тесты их создают

> [!tip] Хорошее правило для AQA
> Если тест работает с файлами, думай о трёх вещах: откуда файл берётся, куда он будет сохранён и кто его очистит после теста.

## Вопросы для собеседования

### Топ-10

**1. Что такое File I/O в Java?**  
Это чтение данных из файлов и запись данных в файлы или другие внешние источники.

**2. В чём разница между `Path` и `File`?**  
`File` — старый API. `Path` — современный NIO.2 API, который лучше работает вместе с utility-методами `Files`.

**3. Что делает `Files.readString()`?**  
Считывает весь текстовый файл в одну строку `String`.

**4. В чём разница между байтовыми и символьными потоками?**  
Байтовые потоки работают с сырыми байтами, обычно для бинарных данных. Символьные потоки работают с текстовыми данными.

**5. Почему try-with-resources важен при работе с файлами?**  
Он автоматически закрывает readers, writers и streams, делая код безопаснее и короче.

**6. Какое исключение часто встречается в файловых операциях?**  
Очень часто встречается `IOException` и его подклассы.

**7. Что проверяет `Files.exists(path)`?**  
Проверяет, существует ли путь на диске.

**8. Как добавить текст в конец файла?**  
Использовать `Files.writeString()` с `StandardOpenOption.APPEND` и обычно `CREATE`.

**9. Где обычно хранят тестовые данные в Java test project?**  
Часто в `src/test/resources`.

**10. Когда файл читают как байты, а не как текст?**  
Когда файл бинарный: изображение, PDF, zip или другой не-текстовый контент.

---

### Хитрые вопросы

**1. Подходит ли `Files.readAllLines()` для очень больших файлов?**  
Не всегда. Он загружает все строки в память. Для очень больших файлов безопаснее буферизованное чтение.

**2. Почему путь к файлу может работать на одной машине и падать на другой?**  
Из-за разных рабочих директорий, разделителей, прав доступа или отсутствующих файлов.

**3. Может ли `Files.writeString()` создать файл?**  
Да, в зависимости от поведения метода и опций. Но родительские директории всё равно должны существовать.

**4. В чём разница между `Files.createDirectory()` и `Files.createDirectories()`?**  
`createDirectory()` создаёт одну директорию и падает, если нет родителя. `createDirectories()` создаёт и недостающие родительские директории.

**5. Почему хардкодить абсолютные пути в тестах обычно плохо?**  
Потому что такие тесты становятся зависимыми от окружения и ломаются на других машинах или CI-серверах.
