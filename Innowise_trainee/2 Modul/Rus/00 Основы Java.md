## Что такое Java

Java — высокоуровневый объектно-ориентированный язык. Главная особенность — платформенная независимость: код работает на любом устройстве с JVM.

### Как это работает

1. Вы пишете `.java` файлы
2. **Компилятор** (`javac`) создаёт `.class` файлы с байткодом
3. **JVM** преобразует байткод в машинный код

> [!info] WORA (Write Once, Run Anywhere)
> Байткод одинаков на всех платформах. Нужна только своя JVM под Windows, Linux или macOS.

## JDK vs JRE vs JVM

| Компонент | Что включает |
|---|---|
| **JVM** | Исполняет байткод. Ядро Java |
| **JRE** | JVM + стандартные библиотеки |
| **JDK** | JRE + инструменты разработки (`javac`) |

> [!tip] Простое правило
> Чтобы запускать → JRE. Чтобы писать и компилировать → JDK. JDK уже включает JRE.

## Первая программа на Java

```java
public class HelloWorld {
    public static void main(String[] args) {
        System.out.println("Hello, World!");
    }
}
```

**Компиляция и запуск:**
```bash
javac HelloWorld.java   # → HelloWorld.class
java HelloWorld         # → Hello, World!
```

## Пакеты и импорт

**Пакет** — папка для группировки классов. Предотвращает конфликты имён.

```java
package com.innowise.training;
import java.util.Scanner;
```

**Соглашение:** обратное доменное имя. Для Innowise: `com.innowise.training.module2`.

## Соглашения об именах

| Что | Стиль | Пример |
|---|---|---|
| Классы | PascalCase | `HelloWorld`, `LoginPage` |
| Методы | camelCase | `getUserName()` |
| Переменные | camelCase | `totalCount` |
| Константы | UPPER_SNAKE | `MAX_SIZE` |
| Пакеты | lowercase | `com.innowise` |

```java
public class UserAccount {
    private static final int MAX_LOGIN_ATTEMPTS = 3;
    private String userName;

    public String getUserName() { return userName; }
}
```

## Вопросы для собеседования

### Топ-10

**1. В чём разница между JDK, JRE и JVM?**
JVM исполняет байткод — это ядро Java. JRE = JVM + стандартные библиотеки (нужна для запуска). JDK = JRE + инструменты разработки (`javac`, `jar`) — нужен для написания и компиляции. JDK включает JRE.

**2. Что означает Write Once, Run Anywhere?**
Java-код компилируется в байткод (`.class`). Байткод не зависит от платформы — он работает на любом устройстве с JVM. Только JVM нужно менять под разные ОС. Код пишешь один раз — работает везде.

**3. Почему main объявлен как `public static void`?**
`public` — JVM должна иметь доступ извне. `static` — JVM вызывает метод без создания объекта (объекта ещё нет при запуске). `void` — main ничего не возвращает JVM.

**4. Чем отличается байткод от машинного кода?**
Байткод — промежуточное представление, не привязанное к конкретному процессору. Машинный код — нативные инструкции для конкретной архитектуры (x86, ARM). JVM преобразует байткод в машинный код через JIT-компилятор.

**5. Что такое пакет в Java?**
Пакет — пространство имён, которое организует классы. Предотвращает конфликты имён, управляет доступом (default modifier). Именуется обратным доменным именем: `com.innowise.training`.

**6. Какие соглашения об именах в Java?**
Классы — PascalCase (`HelloWorld`). Методы и переменные — camelCase (`getUserName`, `totalCount`). Константы — UPPER_SNAKE_CASE (`MAX_SIZE`). Пакеты — строчные (`com.innowise`).

**7. Что произойдёт, если не указать пакет?**
Класс попадёт в *default package*. Для примеров нормально, но в реальных проектах не рекомендуется — могут возникнуть конфликты имён.

**8. Как Java достигает платформенной независимости?**
Через байткод и JVM. Один и тот же `.class` файл работает на любой платформе с JVM. JVM абстрагирует ОС и оборудование.

**9. Что такое `String[] args` в main?**
Массив строк с аргументами командной строки. При запуске `java MyApp arg1 arg2` в `args[0]` будет `"arg1"`, в `args[1]` — `"arg2"`.

**10. Может ли в одном `.java` файле быть несколько public классов?**
Нет. Максимум один public класс, и имя файла должно совпадать с именем этого класса. Можно иметь несколько non-public классов в одном файле.

### Хитрые вопросы

**1. Можно ли сделать main метод не public?**
Скомпилируется — да. Но JVM не найдёт его при запуске, будет `NoSuchMethodError`. JVM ищет конкретно `public static void main(String[])`.

**2. Можно ли скомпилировать и запустить Java-код без main?**
Скомпилировать — да. Запустить — нет. JVM нужна точка входа. Но класс можно запустить через JUnit или как зависимость другого класса.

**3. Чем отличаются `javac` и `java`?**
`javac` — компилятор: переводит `.java` → `.class`. `java` — запускатор: стартует JVM и загружает main-класс.

**4. Почему Java использует и компилятор, и интерпретатор?**
Компилятор (`javac`) создаёт портативный байткод. JVM сначала интерпретирует байткод, а для горячих участков включает JIT-компиляцию в машинный код. Гибрид даёт и переносимость, и производительность.

**5. В чём разница между JIT-компилятором и AOT-компилятором?**
JIT (Just-In-Time) компилирует байткод в машинный код во время выполнения, только для часто используемых методов. AOT (Ahead-Of-Time) компилирует всё заранее (`graalvm native-image`). JIT даёт лучшую оптимизацию, AOT — быстрый старт.
