## if-else

```java
int age = 18;
if (age >= 18) {
    System.out.println("Можно голосовать");
}

int temperature = 30;
if (temperature > 25) {
    System.out.println("Жарко");
} else {
    System.out.println("Не жарко");
}

int score = 85;
if (score >= 90) {
    System.out.println("A");
} else if (score >= 75) {
    System.out.println("B");
} else {
    System.out.println("F");
}
```

> [!tip] Фигурные скобки
> Всегда используй `{}` даже для одной строки.

## switch

```java
int day = 3;
String dayName;
switch (day) {
    case 1: dayName = "Пн"; break;
    case 2: dayName = "Вт"; break;
    case 3: dayName = "Ср"; break;
    case 4: dayName = "Чт"; break;
    case 5: dayName = "Пт"; break;
    case 6:
    case 7: dayName = "Вых"; break;
    default: dayName = "?";
}
```

### Улучшенный switch (Java 14+)

```java
String dayType = switch (day) {
    case 1, 2, 3, 4, 5 -> "Будний";
    case 6, 7 -> "Выходной";
    default -> "?";
};
```

## Цикл for

```java
for (int i = 0; i < 5; i++) {
    System.out.println("Счёт: " + i);
}

// Обратный отсчёт
for (int i = 5; i > 0; i--) { }

// Шаг 2
for (int i = 0; i <= 10; i += 2) { }

// For-each (когда индекс не нужен)
int[] numbers = {10, 20, 30, 40};
for (int num : numbers) {
    System.out.println(num);
}
```

## while и do-while

```java
// while — проверка ДО, может не выполниться
int i = 0;
while (i < 5) { System.out.println(i); i++; }

// do-while — проверка ПОСЛЕ, выполняется хотя бы раз
int j = 0;
do { System.out.println(j); j++; } while (j < 5);
```

## break и continue

```java
// break — выход из цикла
for (int num : numbers) {
    if (num < 0) break;     // остановиться на первом отрицательном
}

// continue — пропустить итерацию
for (int i = 0; i <= 10; i++) {
    if (i % 2 != 0) continue;  // пропустить нечётные
    System.out.println(i);       // 0 2 4 6 8 10
}
```

## Проход по массивам

```java
int[] scores = {85, 90, 78, 92, 88};

for (int i = 0; i < scores.length; i++) {
    System.out.println("Оценка " + i + ": " + scores[i]);
}

int sum = 0;
for (int score : scores) { sum += score; }
double average = (double) sum / scores.length;
```

## Вопросы для собеседования

### Топ-10

**1. В чём разница между for и while?**
`for` — когда известно количество итераций. `while` — когда итерации зависят от условия, которое меняется во время выполнения.

**2. В чём разница между break и continue?**
`break` — немедленный выход из цикла. `continue` — пропуск текущей итерации, переход к следующей.

**3. Чем switch отличается от if-else?**
`switch` удобнее при сравнении одной переменной с конкретными значениями. `if-else` лучше для диапазонов, булевых условий и сложных выражений.

**4. Что такое fall-through в switch?**
Когда case не заканчивается `break`, выполнение "проваливается" в следующий case. Иногда полезно (группировка случаев), но чаще — баг.

**5. Что такое улучшенный switch (Java 14+)?**
Использует стрелку (`->`), не требует `break`. Может использоваться как выражение: `String result = switch (x) { case 1 -> "one"; default -> "other"; };`.

**6. Что такое тернарный оператор? Когда его использовать?**
`условие ? значение_если_true : значение_если_false`. Компактная замена if-else для простых условий. Не злоупотребляй — вложенные тернарки плохо читаются.

**7. Чем отличаются while и do-while?**
`while` проверяет условие ДО тела — может не выполниться ни разу. `do-while` проверяет ПОСЛЕ — выполняется хотя бы раз.

**8. Как работает for-each? Можно ли изменять элементы?**
For-each проходит по массиву или коллекции без индекса. Для массивов — переменная это копия элемента. Для коллекций — можно менять свойства объектов, но нельзя добавлять/удалять элементы (будет `ConcurrentModificationException`).

**9. Что такое бесконечный цикл?**
Цикл, работающий вечно: `while (true) { }` или `for (;;) { }`. Используется в серверных циклах, ожидании событий с break-условием.

**10. Что такое labeled break?**
`break` с меткой выходит из указанного внешнего цикла: `outer: for (...) { for (...) { break outer; } }`. Обычный break выходит только из самого внутреннего цикла.

### Хитрые вопросы

**1. Что выведет `for (int i=0; i<5; i++) { if (i==3) break; System.out.print(i); }`?**
`012`. Цикл останавливается на `i == 3` из-за `break`.

**2. Что выведет `for (int i=0; i<5; i++) { if (i==3) continue; System.out.print(i); }`?**
`0124`. `continue` пропускает `3`, но цикл продолжается.

**3. Что будет, если пропустить break в switch?**
Выполнение провалится в следующий case. Например, если для `case 1: f();` нет `break`, выполнится и то, что в `case 2`.

**4. Можно ли использовать переменную цикла for после его завершения?**
Нет. Переменная объявлена в блоке цикла и недоступна снаружи. Объяви её до цикла, если нужна после.

**5. Чем отличается `++i` от `i++` в шаге цикла for?**
В стандартном for — никакой разницы. Разница проявляется, когда инкремент используется в выражении: `while (i++ < 5)` vs `while (++i < 5)`.
