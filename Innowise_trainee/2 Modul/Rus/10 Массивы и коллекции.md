## Почему эта тема важна

В Java часто нужно хранить не одно значение, а много: список логинов, набор уникальных email, карту тестовых данных или очередь задач. Для этого используются массивы и коллекции.

Эта тема важна, потому что ты будешь использовать её везде:

- В прикладном коде для хранения и обработки данных
- В AQA для локаторов, тестовых данных, API payload и результатов
- На собеседованиях, чтобы объяснить, какая структура подходит для какой задачи

Главная идея простая:

- **Массивы** — контейнеры фиксированного размера
- **Коллекции** — гибкие готовые структуры данных из Java Collections Framework

## Массивы

**Массив** хранит несколько значений одного типа в одном объекте. Его размер фиксируется при создании.

```java
int[] numbers = new int[3];
numbers[0] = 10;
numbers[1] = 20;
numbers[2] = 30;

String[] names = {"Ann", "Bob", "Kate"};
System.out.println(names[1]); // Bob
```

### Ключевые правила

- Индексация начинается с `0`
- Размер нельзя изменить после создания
- Массивы могут хранить примитивы и объекты
- У всех элементов есть значение по умолчанию

```java
int[] nums = new int[3];        // [0, 0, 0]
String[] words = new String[2]; // [null, null]
```

### Преимущества

- Очень быстрый доступ по индексу: `arr[i]`
- Простой синтаксис
- Хороший выбор, когда размер известен заранее

### Недостатки

- Фиксированный размер
- Нет готовых методов вроде `add()` или `remove()`
- Вставка в середину неудобна, потому что элементы нужно сдвигать

> [!tip] Array vs Collection
> Используй массив, когда количество элементов фиксировано и нужен простой доступ по индексу. Используй коллекцию, когда размер часто меняется или нужны готовые операции вроде `add`, `remove`, `contains` или сортировки.

## Обзор Collection Framework

**Java Collections Framework** — это набор интерфейсов и классов для хранения групп объектов.

Основные интерфейсы:

- `List` - упорядоченная коллекция, дубликаты разрешены
- `Set` - только уникальные элементы, без дубликатов
- `Queue` - элементы обрабатываются в определённом порядке
- `Map` - пары ключ-значение

Важно: `Map` является частью framework, но **не** наследует `Collection`.

Популярные реализации:

- `ArrayList`
- `LinkedList`
- `HashSet`
- `HashMap`
- `PriorityQueue`

```java
List<String> list = new ArrayList<>();
Set<String> set = new HashSet<>();
Map<String, Integer> map = new HashMap<>();
Queue<String> queue = new LinkedList<>();
```

## List Set Queue и Map

### List

`List` сохраняет порядок добавления и допускает дубликаты.

```java
List<String> browsers = new ArrayList<>();
browsers.add("Chrome");
browsers.add("Firefox");
browsers.add("Chrome");

System.out.println(browsers); // [Chrome, Firefox, Chrome]
System.out.println(browsers.get(0)); // Chrome
```

Используй `List`, когда важен порядок или когда повторяющиеся значения — это нормально.

### Set

`Set` хранит только уникальные значения.

```java
Set<String> roles = new HashSet<>();
roles.add("admin");
roles.add("user");
roles.add("admin");

System.out.println(roles); // дубликат "admin" игнорируется
```

Используй `Set`, когда нужна уникальность, например уникальные ID или список посещённых страниц.

### Queue

`Queue` полезна, когда элементы нужно обрабатывать по одному, часто по принципу FIFO.

```java
Queue<String> tasks = new LinkedList<>();
tasks.add("open browser");
tasks.add("login");
tasks.add("logout");

System.out.println(tasks.poll()); // open browser
System.out.println(tasks.poll()); // login
```

Используй `Queue` для пайплайнов, фоновых задач или очередей ожидания.

### Map

`Map` хранит данные в формате `ключ -> значение`.

```java
Map<String, String> user = new HashMap<>();
user.put("name", "Alice");
user.put("role", "admin");

System.out.println(user.get("name")); // Alice
```

Используй `Map`, когда нужно найти значение по ключу, например конфиг, поля JSON или заголовки запроса.

> [!warning] Порядок в HashSet и HashMap
> `HashSet` и `HashMap` не гарантируют порядок обхода. Если важен стабильный порядок, используй `LinkedHashSet` или `LinkedHashMap`.

## ArrayList vs LinkedList

Это две самые частые реализации `List`.

### ArrayList

`ArrayList` основан на динамическом массиве.

- Быстрый доступ по индексу
- Обычно лучший выбор по умолчанию
- Медленные вставки и удаления в середине, потому что элементы сдвигаются

### LinkedList

`LinkedList` состоит из связанных узлов.

- Быстрое добавление и удаление по краям
- Медленный доступ по индексу
- Использует больше памяти, чем `ArrayList`

> [!tip] Быстрое сравнение
> | Структура | Лучше всего подходит для | Слабое место |
> |---|---|---|
> | `ArrayList` | Чтение по индексу, большинство обычных задач | Вставка/удаление в середине |
> | `LinkedList` | Частые добавления/удаления по краям | Медленный `get(index)` |

В реальных проектах и тестах `ArrayList` чаще всего является выбором по умолчанию.

## Шпаргалка по ключевым методам

Это методы, которые ты используешь каждый день. Запомни самые частые, остальные смотри по справке.

### List и ArrayList

| Метод | Что делает |
|---|---|
| `add(e)` | Добавить в конец |
| `add(i, e)` | Вставить по индексу `i`, остальное сдвинуть вправо |
| `get(i)` | Прочитать элемент по индексу `i` |
| `set(i, e)` | Заменить элемент по индексу `i`, возвращает старое значение |
| `remove(int i)` | Удалить **по индексу** |
| `remove(Object o)` | Удалить **по значению** (первое совпадение) |
| `removeIf(pred)` | Удалить все элементы, подходящие под условие |
| `indexOf(o)` | Первый индекс значения, или `-1` |
| `contains(o)` | `true`, если значение есть |
| `size()` | Количество элементов |
| `isEmpty()` | `true`, если размер `0` |
| `clear()` | Удалить всё |

```java
List<String> users = new ArrayList<>(List.of("ann", "bob", "kate"));
users.set(0, "alice");          // [alice, bob, kate]
users.add(1, "dave");           // [alice, dave, bob, kate]
users.removeIf(u -> u.length() > 3); // удалит "alice", "dave", "kate"
System.out.println(users.indexOf("bob")); // 0
```

### Set и HashSet

| Метод | Что делает |
|---|---|
| `add(e)` | Добавить; возвращает `false`, если уже есть |
| `remove(o)` | Удалить значение |
| `contains(o)` | Проверка наличия (очень быстрая) |
| `addAll(c)` | Объединение с другой коллекцией |
| `retainAll(c)` | Оставить только элементы, которые есть и в `c` (пересечение) |
| `removeAll(c)` | Удалить все элементы, найденные в `c` (разность) |

```java
Set<String> a = new HashSet<>(List.of("chrome", "firefox", "edge"));
Set<String> b = new HashSet<>(List.of("firefox", "safari"));
a.retainAll(b);                 // пересечение -> [firefox]
System.out.println(a.add("firefox")); // false, уже есть
```

### Map и HashMap

| Метод | Что делает |
|---|---|
| `put(k, v)` | Добавить или перезаписать; возвращает старое значение |
| `get(k)` | Значение по ключу, или `null` |
| `getOrDefault(k, def)` | Значение, или `def`, если ключа нет |
| `putIfAbsent(k, v)` | Записать, только если ключа ещё нет |
| `containsKey(k)` / `containsValue(v)` | Проверки наличия |
| `remove(k)` | Удалить запись по ключу |
| `keySet()` / `values()` / `entrySet()` | Представления для итерации |
| `computeIfAbsent(k, fn)` | Лениво создать и сохранить значение, если ключа нет |
| `merge(k, v, fn)` | Объединить новое и существующее значение (удобно для подсчёта) |

```java
Map<String, Integer> count = new HashMap<>();
for (String word : List.of("a", "b", "a")) {
    count.merge(word, 1, Integer::sum); // {a=2, b=1}
}
System.out.println(count.getOrDefault("x", 0)); // 0

Map<String, List<String>> byType = new HashMap<>();
byType.computeIfAbsent("ui", k -> new ArrayList<>()).add("loginTest");
```

### Queue

| Метод | Что делает |
|---|---|
| `offer(e)` / `add(e)` | Добавить в хвост |
| `poll()` | Удалить и вернуть голову, или `null`, если пусто |
| `peek()` | Посмотреть голову без удаления, или `null` |

```java
Queue<String> tasks = new LinkedList<>();
tasks.offer("login");
tasks.offer("checkout");
System.out.println(tasks.peek()); // login (не удалён)
System.out.println(tasks.poll()); // login (удалён)
```

### Утилиты Arrays и Collections

| Вызов | Что делает |
|---|---|
| `Arrays.sort(arr)` | Отсортировать массив на месте |
| `Arrays.toString(arr)` | Читаемая строка массива |
| `Arrays.asList(...)` | Список фиксированного размера, связанный с массивом |
| `Collections.sort(list)` | Отсортировать список на месте |
| `Collections.reverse(list)` | Обратный порядок |
| `Collections.max(list)` / `min(list)` | Наибольший / наименьший элемент |
| `Collections.frequency(list, o)` | Сколько раз встречается `o` |

## Итерация и основные операции

По массивам и коллекциям очень часто нужно проходить в цикле.

### For Loop

Подходит, когда нужен индекс.

```java
String[] names = {"Ann", "Bob", "Kate"};
for (int i = 0; i < names.length; i++) {
    System.out.println(i + ": " + names[i]);
}
```

### Enhanced for

Подходит, когда нужны только значения.

```java
List<String> cities = List.of("Minsk", "Warsaw", "Berlin");
for (String city : cities) {
    System.out.println(city);
}
```

### Основные операции

```java
List<String> items = new ArrayList<>();
items.add("A");
items.add("B");
items.remove("A");
System.out.println(items.contains("B")); // true
System.out.println(items.size());        // 1
```

### Утилитный класс Arrays

Класс `Arrays` даёт вспомогательные методы для массивов.

```java
int[] nums = {3, 1, 2};
Arrays.sort(nums);
System.out.println(Arrays.toString(nums)); // [1, 2, 3]
```

### Утилитный класс Collections

Класс `Collections` даёт вспомогательные методы для коллекций.

```java
List<Integer> scores = new ArrayList<>(List.of(30, 10, 20));
Collections.sort(scores);
Collections.reverse(scores);
System.out.println(scores); // [30, 20, 10]
```

> [!caution] ConcurrentModificationException
> Не удаляй элементы из коллекции внутри enhanced `for` обычным способом. Это может привести к `ConcurrentModificationException`. Используй `Iterator`, `removeIf()` или собирай элементы в новый список.

## Массивы и коллекции в AQA

В автотестировании массивы и коллекции встречаются постоянно.

### Типичные примеры

```java
List<String> browsers = List.of("chrome", "firefox", "edge");
Map<String, String> headers = Map.of(
    "Content-Type", "application/json",
    "Authorization", "Bearer token"
);
Set<String> uniqueUserIds = new HashSet<>();
```

### Где они используются

- `List<WebElement>` из Selenium: несколько элементов на странице
- `Map<String, Object>` для API payload или разобранного JSON
- `Set<String>` для проверки уникальности ID, ссылок или значений
- Массивы в параметрах методов, например `String[] args`

### Частые ошибки

- Использовать массив, когда размер часто меняется
- Использовать `HashMap` и ждать стабильного порядка
- Забывать, что `Set` убирает дубликаты
- Обращаться по неверному индексу и получать `ArrayIndexOutOfBoundsException`
- Вызывать `get()` у пустого списка

> [!info] Хорошее правило для AQA
> Если нужна упорядоченная гибкая группа элементов, начинай с `ArrayList`. Если нужна уникальность, используй `HashSet`. Если нужны пары ключ-значение, используй `HashMap`.

## Вопросы для собеседования

### Топ-10

**1. В чём разница между массивом и коллекцией?**  
Массив имеет фиксированный размер и может хранить примитивы или объекты. Коллекция имеет гибкий размер и предоставляет готовые методы вроде `add()`, `remove()` и `contains()`. Коллекции хранят объекты, а не примитивы.

**2. В чём разница между `List` и `Set`?**  
`List` сохраняет порядок и допускает дубликаты. `Set` хранит только уникальные элементы и обычно не гарантирует порядок, если не использовать специальную реализацию.

**3. В чём разница между `ArrayList` и `LinkedList`?**  
`ArrayList` лучше подходит для чтения по индексу и обычно быстрее в большинстве задач. `LinkedList` лучше подходит для частых вставок и удалений по краям, но хуже для случайного доступа.

**4. Может ли коллекция хранить примитивные типы?**  
Нет. Коллекции работают с объектами. Используй классы-обёртки: `Integer`, `Double`, `Boolean`.

**5. В чём разница между `HashMap` и `HashSet`?**  
`HashMap` хранит пары ключ-значение. `HashSet` хранит только уникальные значения. Внутри `HashSet` построен на основе `HashMap`.

**6. Что будет, если обратиться к неверному индексу массива?**  
Java выбросит `ArrayIndexOutOfBoundsException`.

**7. В чём разница между `size()` и `length`?**  
У массивов используется поле `length`. У коллекций используется метод `size()`.

**8. Сохраняет ли `HashMap` порядок добавления?**  
Нет. Если нужен порядок добавления, используй `LinkedHashMap`.

**9. Может ли `ArrayList` содержать дубликаты и `null`?**  
Да. `ArrayList` допускает дубликаты и может содержать `null`.

**10. Когда использовать `Map`?**  
Когда нужно находить значение по ключу, например поля пользователя, параметры конфига, headers или тестовые данные.

### Хитрые вопросы

**1. Является ли `Map` подтипом `Collection`?**  
Нет. `Map` относится к Java Collections Framework, но не реализует `Collection`.

**2. Почему `ArrayList` чаще лучше `LinkedList` на практике?**  
Потому что многие реальные задачи требуют быстрого чтения, а процессорный кеш хорошо работает с массивами. У `LinkedList` больше накладных расходов на объекты и медленнее доступ по индексу.

**3. Может ли `Set` содержать дубликаты, если `equals()` и `hashCode()` реализованы неправильно?**  
Да. Если у пользовательского класса сломаны `equals()` и `hashCode()`, set на основе хеша может вести себя неправильно.

**4. В чём разница между `remove(int index)` и `remove(Object o)` в `List`?**  
Один вариант удаляет по позиции, другой — по значению. С `List<Integer>` это часто путают: `remove(1)` удаляет элемент по индексу `1`, а не значение `1`.

**5. Почему `Arrays.asList()` может быть опасным?**  
Он возвращает список фиксированного размера, связанный с массивом. Можно менять элементы, но нельзя добавлять или удалять, иначе будет `UnsupportedOperationException`.

### Вопросы на углублённое понимание

**1. Как работает `HashMap`? Что происходит при `put()` и `get()`?**  
`HashMap` хранит данные в массиве бакетов (`Node<K,V>[]`). При `put()` вычисляется `hashCode(key) → индекс бакета`. Если бакет пуст — кладём новый `Node`. Если занят — сравниваем через `equals()`: если ключи совпали — перезаписываем, если нет — цепляем новый Node в связанный список (коллизия). При длине списка > 8 он превращается в красно-чёрное дерево (treeify) для ускорения поиска. `get()` работает так же: хеш → бакет → поиск по списку/дереву через `equals()`.

**2. Как удалить элементы из коллекции во время перебора?**  
Три правильных способа: (1) `Iterator.remove()` — безопасно удаляет текущий элемент, (2) `Collection.removeIf(predicate)` — лаконичный Java 8+ способ, (3) собрать элементы для удаления в отдельный список и вызвать `removeAll()` после цикла. Неправильный: вызов `list.remove()` внутри enhanced for — выбрасывает `ConcurrentModificationException`.

```java
// Правильно: removeIf
list.removeIf(s -> s.length() < 3);

// Правильно: Iterator
Iterator<String> it = list.iterator();
while (it.hasNext()) {
    if (it.next().length() < 3) it.remove();
}
```

**3. Что такое `ConcurrentModificationException`?**  
Исключение возникает, когда коллекция изменяется (добавление/удаление) во время итерации по ней, если итератор не был уведомлён об изменениях. Это механизм **fail-fast**: итератор проверяет счётчик модификаций (`modCount`) на каждой итерации. Если `modCount` изменился — он падает с исключением, а не продолжает работу с некорректными данными.

**4. В чём разница между `Arrays.asList()` и `List.of()`?**  
`Arrays.asList()` возвращает список фиксированного размера, **подкреплённый** исходным массивом: можно менять элементы (через `set()`), но нельзя добавлять/удалять. Изменения в списке отражаются на массиве и наоборот. `List.of()` (Java 9+) возвращает **immutable** список: нельзя ни добавлять, ни удалять, ни менять элементы (`set()` кидает `UnsupportedOperationException`). И `List.of()` не допускает `null`, а `Arrays.asList()` — допускает.

**5. Как отсортировать `HashMap` по значениям?**  
`HashMap` сам по себе не сортируется. Нужно достать `entrySet()`, скопировать в `List`, отсортировать через `Collections.sort()` или `List.sort()` с компаратором по `Map.Entry.getValue()`, и работать с результатом как со списком.

```java
Map<String, Integer> map = new HashMap<>();
// ... заполнение ...
List<Map.Entry<String, Integer>> entries = new ArrayList<>(map.entrySet());
entries.sort(Map.Entry.comparingByValue());
// entries теперь отсортирован по возрастанию значений
```

**6. В чём разница между `Comparable` и `Comparator`?**  
`Comparable` — естественный порядок (`int compareTo(T o)`), определяется внутри самого класса. Одна сортировка по умолчанию. Пример: `String` сортируется лексикографически.  
`Comparator` — внешняя стратегия сравнения. Можно сделать несколько разных компараторов для одного класса. Используется через `Collections.sort(list, comparator)` или `list.sort()`.

```java
// Comparable — внутри класса:
public class User implements Comparable<User> {
    public int compareTo(User other) { return this.name.compareTo(other.name); }
}

// Comparator — внешний:
Comparator<User> byAge = Comparator.comparingInt(User::getAge);
users.sort(byAge);
```

**7. Что такое `load factor` в `HashMap`?**  
`load factor` (по умолчанию 0.75) определяет, когда увеличивать размер HashMap. Когда количество записей превышает `capacity × load factor`, HashMap увеличивается вдвое (rehash) — пересчитывает хеши и перераспределяет все элементы по новым бакетам. 0.75 — компромисс между памятью и скоростью. Низкое значение (0.5) даёт меньше коллизий, но больше памяти. Высокое (0.9) — экономит память, но растёт время поиска.

**8. Разница между `ArrayList` и `LinkedList` на уровне памяти?**  
`ArrayList` — один непрерывный массив объектов. Занимает меньше памяти: только массив ссылок + данные. Каждый элемент — это ссылка (4–8 байт).  
`LinkedList` — каждый элемент это отдельный `Node`-объект: сам элемент + ссылка на предыдущий + ссылка на следующий (т.е. 3–4 объекта на элемент вместо одного). Для 1 млн элементов `ArrayList` будет занимать ~4–8 MB, а `LinkedList` — ~24–40 MB (в 5–10 раз больше).

**9. Почему для `HashSet` и `HashMap` важно правильно реализовать `equals()` и `hashCode()`?**  
`HashSet` внутри использует `HashMap`. При добавлении сначала вызывается `hashCode()` — он определяет номер бакета. Если два объекта равны (`equals()`), их `hashCode()` **обязан** быть одинаковым (иначе они попадут в разные бакеты, и Set будет считать их разными — дубликат просочится). Обратное не обязательно: разные объекты могут иметь одинаковый хеш (коллизия). Контракт: если `equals()` вернул `true`, то `hashCode()` обязан быть одинаковым.

```java
// Некорректная реализация — Set пропустит дубликаты:
public class User {
    String name;
    public boolean equals(Object o) { /* правильная проверка по name */ }
    // hashCode() не переопределён — у каждого объекта свой hashCode от Object
}
```

**10. Что такое fail-safe итератор?**  
**Fail-fast** (стандартные коллекции: `ArrayList`, `HashMap`, `HashSet`) — падает с `ConcurrentModificationException`, если коллекция изменяется во время итерации.  
**Fail-safe** (коллекции из `java.util.concurrent`: `CopyOnWriteArrayList`, `ConcurrentHashMap`) — создаёт копию данных на момент создания итератора и работает с ней, игнорируя последующие изменения. Минус: не гарантирует актуальности данных и больше памяти.

| Тип | Примеры | Поведение при изменении |
|---|---|---|
| Fail-fast | `ArrayList`, `HashMap`, `HashSet` | `ConcurrentModificationException` |
| Fail-safe | `CopyOnWriteArrayList`, `ConcurrentHashMap` | Работает со снапшотом |

**11. Разница между `HashMap` и `ConcurrentHashMap`?**  
`HashMap` — не синхронизирован, не thread-safe. В многопоточной среде ConcurrentModificationException или гонки данных.  
`ConcurrentHashMap` — thread-safe, использует сегментную блокировку (locking на уровне бакета, не всей карты). Позволяет конкурентное чтение без блокировок, а запись — только на уровне конкретного сегмента. Не кидает `ConcurrentModificationException` при итерации. `null` ключи запрещены.

**12. Разница между `Collection` и `Collections`?**  
`Collection` — корневой интерфейс иерархии коллекций (`List`, `Set`, `Queue`).  
`Collections` — утилитный класс со статическими методами: `sort()`, `reverse()`, `unmodifiableList()`, `synchronizedList()`, `max()`, `min()`, `frequency()`, `emptyList()`, `singleton()` и т.д.

**13. Как растёт `ArrayList` внутри?**  
При создании с дефолтным конструктором создаётся пустой массив. При первом `add()` — массив расширяется до `DEFAULT_CAPACITY = 10`. При заполнении — `grow()`: новый capacity = `oldCapacity + (oldCapacity >> 1)` (в 1.5 раза). Создаётся новый массив, элементы копируются через `Arrays.copyOf()` (нативный System.arraycopy). Операция O(n), но амортизированно O(1) для одного add.

**14. `HashSet` vs `TreeSet`? Когда что использовать?**  
`HashSet` — O(1) add/remove/contains, не гарантирует порядок, требует правильного `hashCode()`.  
`TreeSet` — O(log n), хранит элементы **отсортированными** (по `Comparable` или переданному `Comparator`). Не требует hashCode/equals (использует compareTo). Не допускает `null`.  
Используй `HashSet`, когда нужна только уникальность и скорость. Используй `TreeSet`, когда нужен отсортированный набор (диапазоны, автокомплит, sorted-запросы).

**15. Как работает `PriorityQueue`?**  
Основана на куче (min-heap по умолчанию — бинарное дерево, где каждый родитель меньше потомков). `add()` / `offer()` — кладёт элемент на место и всплывает (O(log n)). `poll()` — забирает корень (минимальный) и просеивает кучу (O(log n)). `peek()` — просто смотрит корень (O(1)). Используется в задачах вроде Top K, планировщики задач, Dijkstra.

**16. Для чего нужен `WeakHashMap`?**  
Ключи в WeakHashMap — слабые ссылки (`WeakReference`). Если на ключ не осталось сильных ссылок вне карты, GC может удалить ключ, и запись автоматически исчезнет из карты при следующей чистке. Используется для кэшей, где нужно, чтобы записи умирали без внешнего использования, или в слушателях событий (уход слушателя → автоматическое удаление из регистрации).

**17. `ArrayDeque` vs `LinkedList` для Deque?**  
`ArrayDeque` — массив с циклическим буфером. Быстрее, меньше памяти, нет объектов Node на каждый элемент. Лучший выбор для стека и очереди.  
`LinkedList` — каждый элемент это Node (prev/next/item). Памяти в 2-3 раза больше. ArrayDeque **всегда** предпочтительнее, если не нужны вставки/удаления в середине (LinkedList это умеет, Deque — нет).

```java
// Стек — ArrayDeque (а не Stack):
Deque<String> stack = new ArrayDeque<>();
stack.push("a"); stack.pop();

// Очередь — ArrayDeque (а не LinkedList):
Queue<String> queue = new ArrayDeque<>();
queue.offer("a"); queue.poll();
```

**18. Как создать immutable коллекцию в Java?**  
Три способа:  
1. `List.of("a", "b")` / `Set.of("a")` / `Map.of("k", "v")` — Java 9+, null-недопустимые, часто оптимизированы.  
2. `Collections.unmodifiableList(list)` — обёртка над существующей коллекцией. Изменения в оригинале **видны** через unmodifiable-представление.  
3. `List.copyOf(list)` / `Set.copyOf()` — создаёт immutable копию (Java 10).  

```java
List<String> mutable = new ArrayList<>();
mutable.add("a");
List<String> unmod = Collections.unmodifiableList(mutable);
mutable.add("b"); // unmod тоже увидит "b"!

List<String> copy = List.copyOf(mutable); // это снапшот
```

**19. Для чего нужны `EnumMap` и `EnumSet`?**  
Очень быстрые (O(1)) и компактные коллекции для enum-ключей.  
`EnumMap` — хранит массив, проиндексированный ordinal() enum. Быстрее HashMap, не считает hashCode.  
`EnumSet` — битовая маска внутри `long[]`. Экстремально быстрый и экономный.  
Используй, когда ключ — enum и нужна максимальная производительность.

```java
enum Status { ACTIVE, INACTIVE, BANNED }
Map<Status, List<User>> usersByStatus = new EnumMap<>(Status.class);
Set<Status> activeStatuses = EnumSet.of(Status.ACTIVE, Status.INACTIVE);
```

**20. Почему не стоит использовать `Stack`? Что вместо него?**  
`Stack` — устаревший класс, наследует `Vector` (синхронизирован, но медленнее). Нарушает LSP: Stack «является» Vector, что позволяет вызывать методы List (get, add по индексу).  
Вместо Stack используй `ArrayDeque` с методами `push()/pop()` для LIFO. ArrayDeque быстрее, легче, не синхронизирован без необходимости.

```java
// ❌ Устарело:
Stack<String> stack = new Stack<>();

// ✅ Правильно:
Deque<String> stack = new ArrayDeque<>();
stack.push("a"); stack.pop();
```

### Вопросы с кодом

**1. Что выведет этот код?**  
```java
List<String> list = new ArrayList<>();
list.add("A"); list.add("B"); list.add("C");
for (String s : list) {
    if (s.equals("B")) list.remove(s);
}
System.out.println(list);
```
**Ответ:** `ConcurrentModificationException`. Enhanced for неявно использует итератор, а `list.remove()` изменяет коллекцию без уведомления итератора. Правильно: `list.removeIf(s -> s.equals("B"))`.

**2. Что выведет этот код?**  
```java
Set<Short> set = new HashSet<>();
for (short i = 0; i < 10; i++) {
    set.add(i);
    set.remove(i - 1); // (int) i - 1
}
System.out.println(set.size());
```
**Ответ:** 10. Ошибка в том, что `i - 1` — это `int`, а не `short`. `set.remove(Integer.valueOf(i-1))` ищет `Integer`, а в Set хранятся `Short`. Ничего не удаляется.

**3. Что выведет этот код?**  
```java
Map<String, String> map = new HashMap<>();
map.put("a", "1");
map.put("b", "2");
map.put("c", "3");
map.computeIfAbsent("a", k -> "10");
map.computeIfAbsent("d", k -> "40");
System.out.println(map.get("a") + map.get("d"));
```
**Ответ:** "140". `computeIfAbsent` выполняется только если ключа нет. `"a"` уже есть — ничего не меняется. `"d"` нет — добавляется `"40"`.

**4. Что выведет этот код?**  
```java
List<Integer> list = new ArrayList<>(List.of(1, 2, 3, 4, 5));
list.subList(1, 3).clear();
System.out.println(list);
```
**Ответ:** `[1, 4, 5]`. `subList()` возвращает **представление** оригинального списка, а не копию. `clear()` на subList удаляет эти элементы из оригинального списка.

**5. Что выведет этот код?**  
```java
Map<String, Integer> map = new HashMap<>();
map.put("x", 1);
map.put("y", 2);
map.merge("x", 10, Integer::sum);
map.merge("z", 30, Integer::sum);
System.out.println(map.get("x") + " " + map.get("z"));
```
**Ответ:** `11 30`. `merge()` для существующего ключа применяет функцию (1 + 10 = 11). Для отсутствующего — просто вставляет значение 30.

### 1. Reverse Array — развернуть массив

**Условие:** Дан массив целых чисел. Разверни его **на месте** (in-place), не используя дополнительный массив.

```java
public void reverse(int[] arr) {
    int left = 0, right = arr.length - 1;
    while (left < right) {
        int tmp = arr[left];
        arr[left] = arr[right];
        arr[right] = tmp;
        left++;
        right--;
    }
}
```

**Разбор:** Техника двух указателей. Один с начала, другой с конца — меняем местами и сдвигаемся к центру. Классика для массивов.

| Метрика | Значение |
|---|---|
| Сложность | O(n) время, O(1) память |
| Ключевой приём | Два указателя (left/right) |

### 2. Find Missing Number — найти пропущенное число

**Условие:** Дан массив из `n` чисел в диапазоне `[0, n]`, одно число пропущено. Найди его.

```java
// Решение 1 — через HashSet (для понимания коллекций):
public int missingNumberSet(int[] nums) {
    Set<Integer> set = new HashSet<>();
    for (int n : nums) set.add(n);
    for (int i = 0; i <= nums.length; i++) {
        if (!set.contains(i)) return i;
    }
    return -1;
}

// Решение 2 — математическое (O(1) по памяти):
public int missingNumberMath(int[] nums) {
    int n = nums.length;
    int expected = n * (n + 1) / 2;
    int sum = 0;
    for (int num : nums) sum += num;
    return expected - sum;
}
```

**Разбор:** HashSet даёт O(1) проверку наличия — очевидный подход. Математический вариант через сумму арифметической прогрессии экономнее по памяти, но требует аккуратности с переполнением при больших n.

| Метрика | Значение |
|---|---|
| Сложность | O(n) время, O(n) или O(1) память |
| Ключевая структура | `HashSet<Integer>` или сумма |

### 3. Remove Duplicates — убрать дубликаты из ArrayList

**Условие:** Дан `ArrayList<String>` с дубликатами. Верни новый список только из уникальных значений, сохранив порядок первого появления.

```java
public List<String> removeDuplicates(List<String> input) {
    Set<String> seen = new LinkedHashSet<>(input); // сохраняет порядок
    return new ArrayList<>(seen);
}

// Ручной вариант (чтобы показать логику):
public List<String> removeDuplicatesManual(List<String> input) {
    Set<String> seen = new HashSet<>();
    List<String> result = new ArrayList<>();
    for (String s : input) {
        if (seen.add(s)) {          // add() вернёт false, если уже есть
            result.add(s);
        }
    }
    return result;
}
```

**Разбор:** `LinkedHashSet` — лучшее из двух миров: уникальность HashSet + порядок ArrayList. Второй вариант показывает механику: проверяем, добавляли ли уже такой элемент.

| Метрика | Значение |
|---|---|
| Сложность | O(n) время, O(n) память |
| Ключевые структуры | `LinkedHashSet<T>`, `HashSet<T>` + `ArrayList<T>` |

### 4. Merge Two Sorted Lists — слить два отсортированных списка

**Условие:** Даны два отсортированных `List<Integer>`. Верни новый отсортированный список, содержащий все элементы из обоих.

```java
public List<Integer> mergeSorted(List<Integer> a, List<Integer> b) {
    List<Integer> result = new ArrayList<>(a.size() + b.size());
    int i = 0, j = 0;

    while (i < a.size() && j < b.size()) {
        result.add(a.get(i) <= b.get(j) ? a.get(i++) : b.get(j++));
    }
    while (i < a.size()) result.add(a.get(i++));
    while (j < b.size()) result.add(b.get(j++));

    return result;
}
```

**Разбор:** Техника двух указателей на ArrayList. Меньший элемент забираем, указатель сдвигаем. Когда один список кончился — дописываем хвост второго. Метод `get(i)` у ArrayList работает за O(1).

| Метрика | Значение |
|---|---|
| Сложность | O(a + b) время, O(a + b) память |
| Ключевые структуры | `ArrayList<Integer>` + два указателя |

### 5. Rotate List — сдвинуть ArrayList на k позиций

**Условие:** Дан `ArrayList<Integer>` и число `k`. Сдвинь все элементы вправо на `k` позиций. Последние k элементов становятся первыми.

```java
public void rotate(List<Integer> list, int k) {
    int n = list.size();
    if (n == 0) return;
    k = k % n;             // если k больше размера
    if (k == 0) return;

    // Разворачиваем весь список, потом части
    Collections.reverse(list);           // [7,6,5,4,3,2,1] для n=7,k=3
    Collections.reverse(list.subList(0, k));   // первые k
    Collections.reverse(list.subList(k, n));   // остальные
    // Результат: [5,6,7,1,2,3,4]
}
```

**Разбор:** Тройной reverse — элегантный трюк, популярный на собесах. Работает за O(n) и in-place (не создаёт новый список). `Collections.reverse()` + `subList()` — полезная комбинация при работе с ArrayList.

| Метрика | Значение |
|---|---|
| Сложность | O(n) время, O(1) память |
| Ключевые методы | `Collections.reverse()`, `List.subList()` |

### 6. Find Middle — найти середину списка

**Условие:** Дан `List<Integer>`. Верни элемент, находящийся в середине. Для чётного размера — левый из двух центральных.

```java
// Простой способ (только для ArrayList):
public int findMiddleSimple(List<Integer> list) {
    return list.get((list.size() - 1) / 2);
}

// Универсальный способ (два указателя, работает с LinkedList):
public int findMiddleTwoPointer(List<Integer> list) {
    int slow = 0, fast = 0;
    while (fast < list.size() - 1) {
        slow++;           // медленный — шаг 1
        fast += 2;        // быстрый — шаг 2
    }
    return list.get(slow);
}
```

**Разбор:** `get(index)` у ArrayList — O(1), так что ответ в лоб. Но техника двух указателей (fast/slow) обязательна к знанию: она работает для односвязных списков (LinkedList), где нет быстрого доступа по индексу.

| Метрика | Значение |
|---|---|
| Сложность | O(1) / O(n) время, O(1) память |
| Ключевой приём | Два указателя (fast / slow) |

### 7. Intersection of Two Arrays — пересечение через Set

**Условие:** Даны два массива. Верни их пересечение — элементы, которые встречаются в обоих, без дубликатов.

```java
public Set<Integer> intersection(int[] a, int[] b) {
    Set<Integer> setA = new HashSet<>();
    for (int n : a) setA.add(n);

    Set<Integer> result = new HashSet<>();
    for (int n : b) {
        if (setA.contains(n)) {
            result.add(n);
        }
    }
    return result;
}

// Компактнее через retainAll (в одну строку):
public Set<Integer> intersectionRetain(int[] a, int[] b) {
    Set<Integer> setA = Arrays.stream(a).boxed().collect(Collectors.toSet());
    Set<Integer> setB = Arrays.stream(b).boxed().collect(Collectors.toSet());
    setA.retainAll(setB);
    return setA;
}
```

**Разбор:** `retainAll()` — специальный метод Set для пересечения. Сначала собираем один массив в HashSet (даёт O(1) проверку), потом фильтруем второй.

| Метрика | Значение |
|---|---|
| Сложность | O(a + b) время, O(min(a, b)) память |
| Ключевая структура | `HashSet<Integer>` + `retainAll()` |

### 8. Two Sum — найти индексы двух чисел, дающих target

**Условие:** Дан массив `nums` и `target`. Верни индексы двух чисел, сумма которых равна `target`. Гарантируется ровно одно решение.

```java
public int[] twoSum(int[] nums, int target) {
    Map<Integer, Integer> map = new HashMap<>(); // значение -> индекс
    for (int i = 0; i < nums.length; i++) {
        int complement = target - nums[i];
        if (map.containsKey(complement)) {
            return new int[]{map.get(complement), i};
        }
        map.put(nums[i], i);
    }
    return new int[0]; // не найдено
}
```

**Разбор:** Самая известная задача на HashMap. Проходим массив один раз, на каждом шаге проверяем, не встречали ли уже число-дополнение. O(1) поиск в Map делает решение линейным вместо квадратичного перебора.

| Метрика | Значение |
|---|---|
| Сложность | O(n) время, O(n) память |
| Ключевая структура | `HashMap<Integer, Integer>` |

### 9. Count Character Frequency — частота символов в строке

**Условие:** Дана строка. Верни `Map<Character, Integer>`, где ключ — символ, значение — сколько раз он встречается.

```java
public Map<Character, Integer> charFrequency(String s) {
    Map<Character, Integer> freq = new HashMap<>();
    for (char c : s.toCharArray()) {
        freq.merge(c, 1, Integer::sum);  // лаконичный подсчёт
    }
    return freq;
}

// Понятный вариант (для новичков):
public Map<Character, Integer> charFrequencyClassic(String s) {
    Map<Character, Integer> freq = new HashMap<>();
    for (char c : s.toCharArray()) {
        if (freq.containsKey(c)) {
            freq.put(c, freq.get(c) + 1);
        } else {
            freq.put(c, 1);
        }
    }
    return freq;
}
```

**Разбор:** `merge(c, 1, Integer::sum)` заменяет целый if-else блок. Если ключа нет — вставляет 1, если есть — прибавляет к текущему значению.

| Метрика | Значение |
|---|---|
| Сложность | O(n) время, O(n) память |
| Ключевая структура | `HashMap<Character, Integer>` + `merge()` |

### 10. Most Frequent Element — самый частый элемент

**Условие:** Дан массив целых чисел. Верни элемент, который встречается чаще всех. Если таких несколько — верни любой.

```java
public int mostFrequent(int[] nums) {
    Map<Integer, Integer> freq = new HashMap<>();
    for (int n : nums) {
        freq.merge(n, 1, Integer::sum);
    }
    return Collections.max(freq.entrySet(), Map.Entry.comparingByValue()).getKey();
}
```

**Разбор:** Два шага: подсчёт через HashMap, поиск максимума через `Collections.max()` с компаратором по значению. В production для больших данных лучше PriorityQueue, но на собесе это лаконичное решение проходит.

| Метрика | Значение |
|---|---|
| Сложность | O(n) время, O(n) память |
| Ключевые структуры | `HashMap<Integer, Integer>` + `Collections.max()` |

### 11. Valid Parentheses — правильная скобочная последовательность

**Условие:** Дана строка из скобок `(){}[]`. Определи, правильно ли они открыты, закрыты и вложены.

```java
public boolean isValid(String s) {
    Map<Character, Character> pairs = Map.of(')', '(', '}', '{', ']', '[');
    Deque<Character> stack = new ArrayDeque<>();

    for (char c : s.toCharArray()) {
        if (pairs.containsValue(c)) {           // открывающая скобка
            stack.push(c);
        } else {                                  // закрывающая
            if (stack.isEmpty() || stack.pop() != pairs.get(c)) {
                return false;
            }
        }
    }
    return stack.isEmpty();
}
```

**Разбор:** `Deque` (интерфейс) с `ArrayDeque` — современная замена устаревшему `Stack`. Открывающие скобки кладём, на закрывающих проверяем верхушку. Совпадение не сошлось или стек пуст — ошибка.

| Метрика | Значение |
|---|---|
| Сложность | O(n) время, O(n) память |
| Ключевая структура | `Deque<Character>` (`ArrayDeque`) |

### 12. Top K Frequent Elements — K самых частых элементов

**Условие:** Дан массив целых чисел и `k`. Верни `k` наиболее часто встречающихся элементов.

```java
public List<Integer> topKFrequent(int[] nums, int k) {
    // 1. Подсчёт частоты
    Map<Integer, Integer> freq = new HashMap<>();
    for (int n : nums) freq.merge(n, 1, Integer::sum);

    // 2. Сортировка по убыванию частоты
    List<Map.Entry<Integer, Integer>> entries = new ArrayList<>(freq.entrySet());
    entries.sort((a, b) -> b.getValue() - a.getValue());

    // 3. Первые k элементов
    List<Integer> result = new ArrayList<>();
    for (int i = 0; i < k && i < entries.size(); i++) {
        result.add(entries.get(i).getKey());
    }
    return result;
}
```

**Разбор:** Комбинация всех трёх структур: HashMap для подсчёта, ArrayList для сортировки entrySet() по значениям. В production стоит PriorityQueue (O(n log k) вместо O(n log n)), но эта версия нагляднее для собеса.

| Метрика | Значение |
|---|---|
| Сложность | O(n log n) время, O(n) память |
| Ключевые структуры | `HashMap` → `ArrayList<Entry>` → сортировка |

---

> [!tip] Что запомнить для собеса
> - **HashMap** — для «найди быстро» и «подсчитай»
> - **HashSet** — для уникальности и пересечений
> - **ArrayList** — лучший выбор по умолчанию, get() за O(1)
> - **Deque/ArrayDeque** — для стека и очереди (скобки, палиндромы)
> - **Два указателя** — приём для массивов и списков (reverse, merge, find middle)
> - **Всегда называй O(...)** после решения — показывает глубину понимания
