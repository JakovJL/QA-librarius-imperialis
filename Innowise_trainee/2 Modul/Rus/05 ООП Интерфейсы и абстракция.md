## Что такое интерфейс

**Интерфейс** — это контракт. Определяет **что** класс должен делать, но не **как**.

```java
public interface Drawable {
    void draw();
    double getArea();
}

public class Circle implements Drawable {
    private double radius;
    public Circle(double radius) { this.radius = radius; }

    @Override
    public void draw() { System.out.println("Рисуем круг"); }

    @Override
    public double getArea() { return Math.PI * radius * radius; }
}
```

> [!info] Синтаксис
> - `interface` вместо `class`
> - Методы без тела
> - Поля — только `public static final`
> - Класс использует `implements`

## Реализация интерфейсов

Класс может реализовать несколько интерфейсов.

```java
public interface Printable { void printInfo(); }
public interface Resizable { void resize(double factor); }

public class Document implements Printable, Resizable {
    @Override
    public void printInfo() { System.out.println("Документ"); }

    @Override
    public void resize(double factor) { /* ... */ }
}
```

### Интерфейс как тип

```java
Printable doc = new Document("Hello");
doc.printInfo();   // работает
```

> [!tip] Программируй на уровне интерфейсов
> `List<String> list = new ArrayList<>();` — гибко, можно переключить реализацию.

## Default и статические методы (Java 8+)

```java
public interface Vehicle {
    void start();
    default void honk() { System.out.println("Бип!"); }
}

public class Car implements Vehicle {
    @Override
    public void start() { System.out.println("Машина заведена"); }
    // honk() наследуется автоматически
}
```

## Функциональные интерфейсы

Один абстрактный метод. Основа лямбд.

```java
@FunctionalInterface
public interface Greeting {
    String greet(String name);
}

Greeting hello = name -> "Привет, " + name;
```

### Встроенные

| Интерфейс | Метод | Назначение |
|---|---|---|
| `Predicate<T>` | `boolean test(T t)` | Проверить условие |
| `Consumer<T>` | `void accept(T t)` | Сделать что-то |

## Абстрактные классы vs Интерфейсы

| Особенность | Абстрактный класс | Интерфейс |
|---|---|---|
| Поля | Любые | Только константы |
| Конструкторы | ✅ | ❌ |
| Множественное наследование | ❌ | ✅ |

### Когда что выбирать

Абстрактный класс — общее состояние + частичное поведение.
Интерфейс — контракт/способность.

## Вопросы для собеседования

### Топ-10

**1. Что такое интерфейс в Java?**
Интерфейс — это контракт. Определяет, что класс должен делать, но не как. Методы неявно `public` и `abstract`. Класс реализует интерфейс через `implements`.

**2. В чём разница между абстрактным классом и интерфейсом?**
Абстрактный класс может иметь любые поля, конструкторы и любые методы. Интерфейс — только `public static final` поля, без конструкторов. Класс extends один абстрактный класс, implements много интерфейсов.

**3. Может ли класс реализовать несколько интерфейсов?**
Да. Это способ множественного наследования в Java: `class MyClass implements A, B { }`. Класс должен реализовать все методы из всех интерфейсов.

**4. Что такое default методы в интерфейсах? (Java 8+)**
Default методы — методы с телом, которые можно добавить в интерфейс без поломки существующих реализаций. Классы-реализаторы наследуют их автоматически и могут переопределить при необходимости.

**5. Что такое функциональный интерфейс?**
Интерфейс ровно с одним абстрактным методом. Основа лямбда-выражений. Аннотация `@FunctionalInterface` необязательна, но рекомендуется.

**6. Чем отличается extends для интерфейсов от implements для классов?**
`extends` — интерфейс наследует другой интерфейс: `interface B extends A`. `implements` — класс выполняет контракт интерфейса: `class C implements A`.

**7. Могут ли интерфейсы иметь поля?**
Да, но неявно `public static final` — только константы. Пример: `int MAX = 100;` эквивалентно `public static final int MAX = 100;`.

**8. Что такое проблема ромба (diamond problem) и как Java её решает?**
Если класс реализует два интерфейса с одинаковым default методом, возникает конфликт. Java требует, чтобы класс переопределил этот метод явно. Можно вызвать один из родительских: `InterfaceA.super.methodName()`.

**9. Зачем нужны static методы в интерфейсах? (Java 8+)**
Для утилитных методов, относящихся к концепции интерфейса. Вызываются через имя интерфейса: `Validator.isValidEmail(email)`. Не наследуются классами-реализаторами.

**10. Почему интерфейсы полезны в тестовой автоматизации?**
Позволяют отделить логику тестов от реализации. `WebDriver driver = new ChromeDriver()` — можно переключиться на Firefox без изменения тестов. Page Object через интерфейсы позволяет заменить Selenium на Playwright сменой одного класса.

### Хитрые вопросы

**1. Может ли интерфейс расширять несколько интерфейсов?**
Да. В отличие от классов, интерфейсы могут наследовать несколько: `interface C extends A, B { }`. Это безопасно, так как интерфейсы не содержат реализации.

**2. Может ли default метод быть final?**
Нет. Default методы созданы для переопределения. `final` противоречит их назначению, компилятор не разрешит.

**3. Что будет, если класс реализует два интерфейса с одинаковым default методом?**
Класс обязан переопределить метод явно. Иначе — ошибка компиляции (diamond problem). Можно вызвать один из родительских: `InterfaceA.super.methodName()`.

**4. Можно ли создать экземпляр интерфейса через new?**
Нет. Интерфейсы нельзя инстанциировать. Но можно создать анонимный класс или лямбду (для функциональных интерфейсов), которые реализуют интерфейс на лету.

**5. Могут ли интерфейсы иметь private методы? (Java 9+)**
Да. Private методы в интерфейсах (Java 9+) используются для общего кода между default методами внутри того же интерфейса. Не видны реализующим классам.
