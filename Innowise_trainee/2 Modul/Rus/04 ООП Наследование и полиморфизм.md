## Основы наследования

Потомок переиспользует поля и методы родителя. Ключевое слово — `extends`.

```java
public class Animal {
    protected String name;
    public void eat() { System.out.println(name + " ест"); }
}

public class Dog extends Animal {
    public void bark() { System.out.println(name + " гавкает"); }
}

Dog dog = new Dog();
dog.name = "Rex";
dog.eat();      // унаследовано
dog.bark();     // своё
```

> [!info] Одиночное наследование
> Класс может расширять только одного родителя. Многоуровневость допустима.

## super

```java
public class Vehicle {
    public Vehicle(String brand) { }
}

public class Car extends Vehicle {
    public Car(String brand) {
        super(brand);       // должна быть первой строкой
    }
}
```

## Переопределение (overriding)

Потомок заново определяет метод родителя. Та же сигнатура.

```java
public class Shape {
    public double getArea() { return 0; }
}

public class Circle extends Shape {
    @Override
    public double getArea() { return Math.PI * radius * radius; }
}
```

---

## Перегрузка (overloading)

Одно имя, разные параметры в одном классе.

```java
public class Calculator {
    public int add(int a, int b) { return a + b; }
    public double add(double a, double b) { return a + b; }
}
```

## final

| Использование | Эффект |
|---|---|
| `final` переменная | Константа |
| `final` метод | Нельзя переопределить |
| `final` класс | Нельзя наследовать |

## Абстрактные классы

Нельзя создать напрямую. Могут содержать абстрактные методы (без тела).

```java
public abstract class Payment {
    public abstract void processPayment();
    public void printReceipt() { System.out.println("Платёж обработан"); }
}

public class CreditCardPayment extends Payment {
    @Override
    public void processPayment() {
        System.out.println("Оплата картой");
    }
}
```

---

## Полиморфизм

Один вызов — разное поведение в зависимости от реального типа объекта.

```java
Payment p1 = new CreditCardPayment(100, "1234");
Payment p2 = new PayPalPayment(50, "email");
p1.processPayment();   // версия CreditCardPayment
p2.processPayment();   // версия PayPalPayment
```

## instanceof

Проверяет, является ли объект экземпляром класса.

```java
if (payment instanceof CreditCardPayment) {
    CreditCardPayment cc = (CreditCardPayment) payment;
}
```

> [!caution] Запах кода
> Частое instanceof значит, что дизайн можно улучшить. Предпочитай полиморфизм.

## Вопросы для собеседования

### Топ-10

**1. Что такое наследование в Java? Какое ключевое слово используется?**
Наследование позволяет классу-потомку переиспользовать поля и методы родителя. Ключевое слово — `extends`. Java поддерживает только одиночное наследование (один родитель).

**2. Чем отличается перегрузка (overloading) от переопределения (overriding)?**
Перегрузка — одно имя, разные параметры, в том же классе (compile-time полиморфизм). Переопределение — та же сигнатура, в классе-потомке, заменяет родительский метод (runtime полиморфизм).

**3. Что делает ключевое слово super?**
`super` ссылается на родительский класс. Используется для: (1) вызова конструктора родителя (`super(params)` — должна быть первой строкой), (2) вызова переопределённого метода родителя (`super.methodName()`).

**4. Чем отличается абстрактный класс от конкретного?**
Абстрактный класс (с `abstract`) нельзя создать через `new`. Может содержать абстрактные методы (без тела). Конкретный класс можно инстанциировать, он обязан реализовать все унаследованные абстрактные методы.

**5. Зачем нужна аннотация @Override?**
`@Override` говорит компилятору: "я собираюсь переопределить метод родителя". Если метода с такой сигнатурой нет — ошибка компиляции. Ловит опечатки и неправильные типы параметров.

**6. Что такое полиморфизм?**
"Много форм" — один и тот же вызов метода ведёт себя по-разному в зависимости от реального типа объекта. Пример: `Animal a = new Dog(); a.makeSound();` вызовет версию Dog, а не Animal.

**7. Что делает final для метода? Класса? Переменной?**
`final` метод — нельзя переопределить. `final` класс — нельзя наследовать. `final` переменная — константа, нельзя изменить.

**8. Почему в Java нет множественного наследования классов?**
Из-за "проблемы ромба": если два родителя имеют одинаковый метод, потомок не знает, какую версию наследовать. Решение — интерфейсы: класс может реализовать много интерфейсов.

**9. Что такое instanceof?**
Оператор, проверяющий, является ли объект экземпляром класса (или его наследника). Возвращает `boolean`. Используется перед кастом для безопасного приведения.

**10. Чем отличается extends от implements?**
`extends` — наследование класса (или интерфейса). Класс extends один класс. `implements` — реализация интерфейса. Класс implements много интерфейсов.

### Хитрые вопросы

**1. Может ли конструктор быть final или static?**
Нет. Конструкторы не наследуются и не переопределяются, так что `final` не имеет смысла. `static` — тоже нет, конструктор всегда связан с созданием объекта.

**2. Может ли абстрактный класс иметь конструктор? Зачем?**
Да. Конструктор вызывается через `super()` из конструктора потомка. Нужен для инициализации полей абстрактного класса до того, как выполнится конструктор потомка.

**3. Почему частое использование instanceof — плохой стиль?**
Потому что обходит полиморфизм. Вместо `if (animal instanceof Dog) { ((Dog)animal).bark(); }` нужно `animal.makeSound();`. Частый instanceof — сигнал, что иерархию нужно перепроектировать.

**4. Можно ли вызвать super.super.methodName()?**
Ошибка компиляции. Java не позволяет обращаться к методам "дедушки" напрямую, если родитель их переопределил. Только прямой родитель доступен через `super`.

**5. Можно ли переопределить private метод?**
Нет. Private методы не наследуются. Метод с тем же именем в потомке — новый метод, не override. Если добавить `@Override`, компилятор выдаст ошибку.
